Ad Break Marker
자동화 도구

영상을 AI가 분석해서 광고 넣기 좋은 구간 후보를
Premiere Pro 마커 파일로 자동 생성해 드립니다.

💡 이 도구는 무엇인가요?

광고 타임코드를 일일이 수동으로 찾는 시간을 줄이기 위해 만든 내부 자동화 도구입니다.

📌 왜 만들었나요?

편집자가 광고 삽입 지점을 찾으려면 영상을 처음부터 끝까지 직접 봐야 했습니다. 이 도구는 AI가 "광고 넣기 좋은 구간 후보"를 먼저 좁혀주고, 편집자는 최종 선택만 하면 됩니다.

✅ 두 가지 원칙

  • 완전 자동이 아닙니다. AI는 후보를 제안하고 결정은 편집자가 합니다.
  • 허용 프레임에만 마커를 찍습니다. 방송 기준에 맞는 특정 프레임(:00~:03, :28~:29)에만 마커가 생성됩니다.
ℹ️ 결과물은 Premiere Pro에서 파일 → 가져오기로 불러오는 XML 파일입니다. 불러오면 타임라인에 마커가 자동으로 표시됩니다.

② AI 검증 — 장면이 진짜 바뀐 건지 확인합니다

첫 번째 단계에서 찾은 "장면 전환 후보"들을 AI가 컷 전후 화면 두 장을 비교해 실제로 다른 장면인지 확인합니다. 단순히 카메라 앵글이 바뀌거나 자막·그래픽이 들어온 것은 걸러냅니다. 동시에 컷 전후의 발화 내용을 읽어서 이야기 주제가 바뀌었는지도 파악합니다.

더 자세히 보기

이미지 비교 AI(CLIP)가 컷 전후 프레임을 비교해 유사도 0~1로 수치화합니다. 0.80 미만이면 진짜 장면 전환으로 확정합니다. 한국어 문장 이해 AI(ko-sroberta)는 컷 전후 발화를 비교해 주제 전환 여부를 판단합니다. 이 결과는 마커 점수에 반영됩니다.

③ 후보 생성 — 세 가지 상황에서 마커를 찍습니다

아래 세 가지 상황 중 하나라도 해당되면 마커 후보가 생성됩니다. 중복되는 지점은 자동으로 하나로 합칩니다.

Path 1
말이 끊기는 순간
문장이 마무리되고 잠깐 조용해지는 구간입니다. 강의, 토크쇼, 예능처럼 말이 중심인 콘텐츠에서 가장 많이 잡힙니다.
  • ✓문장이 자연스럽게 끝나고
  • ✓0.5초 이상 음성이 없고
  • ✓그 침묵 안에 허용 프레임이 있을 때
Path 2
화면이 확 바뀌는 순간
AI가 확인한 진짜 장면 전환 지점입니다. 빠르게 편집된 영상에서 침묵이 없어도 잡아냅니다.
  • ✓AI가 진짜 장면 전환으로 확인하고
  • ✓전환 지점 근처에 문장 경계가 있고
  • ✓해당 프레임이 허용 프레임일 때
Path 3
화면이 어두워지는 순간
페이드 아웃으로 암전되는 구간입니다. 영화·음악 콘텐츠처럼 장면 사이에 페이드가 자주 쓰이는 영상에서 효과적입니다.
  • ✓화면이 서서히 어두워지는 구간이고
  • ✓가장 어두운 프레임 근처에 음성이 없고
  • ✓그 침묵 안에 허용 프레임이 있을 때

④ 파일 출력 — 두 종류의 XML이 만들어집니다

1차 XML _adbreaks.xml

간격 규칙이 적용된 실사용 파일입니다. 첫 광고는 3~10분 사이, 이후 광고는 10~15분 간격으로 배치됩니다. 같은 구간 안에서 점수 순으로 최대 5개 후보가 표시됩니다.

2차 XML _adbreaks_all.xml

장면 전환·페이드가 확인된 모든 지점을 간격 제한 없이 담은 파일입니다. 1차 파일을 검토할 때 보조 참고용으로 사용합니다.

자세한 기술 정보 보기

🗣️ 음성 인식 (Whisper)

OpenAI가 만든 음성 인식 AI입니다. 영상의 한국어 음성을 텍스트로 변환하고 각 문장의 시작·끝 시각을 밀리초 단위로 파악합니다. Apple Silicon GPU를 활용해 CPU보다 약 7배 빠르게 처리합니다.

🎬 장면 비교 AI (CLIP)

OpenAI의 이미지 이해 AI입니다. 컷 전후 프레임 두 장을 비교해 같은 장면인지 다른 장면인지를 0~1 유사도로 수치화합니다. 0.80 미만이면 진짜 장면 전환으로 확정합니다.

🧠 한국어 이해 AI (ko-sroberta)

한국어 특화 언어 모델입니다. 두 문장을 읽고 내용이 얼마나 비슷한지 수치화합니다. 0.75 미만이면 주제가 바뀐 것으로 판단해 점수에 반영합니다.

🎥 영상 처리 (ffmpeg / PySceneDetect)

영상 분석 전용 도구입니다. PySceneDetect가 픽셀 변화로 장면 전환 후보를 빠르게 탐지하고, ffmpeg이 음성 음량 곡선과 프레임별 밝기 데이터를 추출합니다.

🔍 픽셀 비교 vs CLIP — 왜 두 가지를 같이 쓰나요?

PySceneDetect(픽셀 비교)는 인접한 두 프레임의 RGB 픽셀 값 차이를 수치로 계산합니다. 전체적으로 픽셀이 많이 바뀌면 장면이 전환됐다고 판단합니다. 계산이 단순해서 빠르지만, 조명이 갑자기 바뀌거나 카메라가 빠르게 흔들려도 장면 전환으로 잘못 잡히는 경우가 있습니다.

CLIP은 이미지와 텍스트를 수억 장 함께 학습해서 "이 화면이 어떤 장면인지"를 이해하는 딥러닝 모델입니다. 픽셀이 많이 달라져도 같은 장소·인물이면 전환이 아닌 것으로 걸러냅니다.

이 프로젝트에서 두 가지를 함께 쓰는 이유가 바로 이겁니다. PySceneDetect로 빠르게 후보를 좁히고, CLIP으로 그 후보가 진짜 장면 전환인지 정밀하게 검증합니다. 속도와 정확도를 동시에 잡는 구조입니다.

📊 어떤 지점을 좋은 후보로 보나요?

조건을 통과한 마커끼리 순위를 정할 때 아래 기준으로 점수를 매깁니다. 점수가 높을수록 1차 XML에서 앞에 추천됩니다.

✅ 이런 지점을 선호해요

● 화면이 다른 장소·장면으로 바뀌는 순간
● 이야기 주제가 완전히 달라지는 전환점
● "자 이제", "다음으로", "정리하자면" 같이 새 내용을 시작하는 말
● "이상입니다", "마치겠습니다" 같이 내용을 마무리하는 말

❌ 이런 지점은 피해요

● 구독, 좋아요, 알림 등 홍보성 멘트가 있는 구간
● "근데", "그리고", "사실" 처럼 말이 계속 이어지는 구간
● 질문을 던지고 바로 답하는 자문자답 중간
● 앞 문장이 너무 짧아서 말 조각처럼 보이는 경우

Premiere Pro 마커 패널에서 최종 마커를 확정한 뒤, 마커 패널 메뉴에서 내보내기 → CSV를 선택합니다.

저장된 파일 이름은 자동으로 YBJ_S25_EP01_AdBreakCandidates.csv 형태가 됩니다.

⚠️ 파일명에서 _AdBreakCandidates 부분을 지우면 안 됩니다. 시트에서 영상 파일명을 자동으로 찾을 때 이 규칙을 사용합니다.
2

구글 시트에서 메뉴 열기

작업시트가 있는 구글 스프레드시트를 열고, 상단 메뉴에서 🎬 AD 마커 → 📥 CSV에서 가져오기를 클릭합니다.

ℹ️ 메뉴가 보이지 않으면 스프레드시트를 새로고침(F5)해 보세요. 처음 열 때 메뉴가 로딩되는 데 몇 초 걸릴 수 있습니다.
3

CSV 파일 선택 후 가져오기

팝업 창이 열리면 파일 선택 버튼을 눌러 1단계에서 내보낸 CSV 파일을 선택합니다. 여러 파일을 한 번에 선택할 수 있습니다.

가져오기 버튼을 누르면 파일명을 기준으로 작업시트 C열을 검색해 해당 행의 F열(AD)에 타임코드가 자동으로 입력됩니다.

4

결과 확인

완료 메시지에서 성공/실패 여부를 확인합니다. 입력된 타임코드는 방송 허용 프레임 기준으로 자동 변환됩니다.

타임코드 변환 규칙

프레임 :00~:03 → 초 버림 (예: 00:09:09:03 → 00:09:09)

프레임 :28~:29 → 초 올림 (예: 00:09:09:28 → 00:09:10)

✅ 한 번에 에피소드 여러 개의 CSV를 선택해도 됩니다. 각 파일명에 맞는 행을 자동으로 찾아서 입력합니다.
