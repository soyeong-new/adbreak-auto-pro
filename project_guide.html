자동화 도구

영상을 AI가 분석해서 광고 넣기 좋은 구간 후보를
Premiere Pro 마커 파일로 자동 생성해 드립니다.

이 도구는 무엇인가요?

영상을 분석해 광고 넣기 좋은 구간 후보를 찾아주는 도구입니다. AI가 후보를 좁혀주고, 최종 선택은 편집자가 합니다.

허용 프레임에만 마커를 찍습니다

마커는 방송 기준에 맞는 특정 프레임에만 생성됩니다. 30fps 기준 :00 / :01 / :02 / :03 / :28 / :29 프레임만 허용하며, 이 외의 위치로는 어떤 경우에도 마커를 만들지 않습니다.

어떻게 작동하나요?

음성 음량 흐름과 전체 음량 흐름은 다른 데이터입니다. 음성 음량은 사람 목소리 대역(250~3000Hz)만 걸러서 "말이 멈췄는지"를 보고, 전체 음량은 대역 구분 없이 전체 소리를 봐서 "배경음악이 깔려 있는지"를 봅니다. 둘 다 조용하면 BGM 없이 완전히 정지된 구간이라는 뜻이라 광고 넣기 더 좋은 지점으로 봅니다.
자세한 기술 정보 보기

음성 인식 (Whisper)

OpenAI가 만든 음성 인식 AI입니다. 영상의 한국어 음성을 텍스트로 변환하고 각 문장의 시작·끝 시각을 밀리초 단위로 파악합니다. Apple Silicon GPU를 활용해 CPU보다 약 7배 빠르게 처리합니다.

영상 처리 (ffmpeg / PySceneDetect)

영상 분석 전용 도구입니다. PySceneDetect가 픽셀 변화로 장면 전환 후보를 빠르게 탐지하고, ffmpeg이 음성 음량 곡선·전체 음량 곡선·프레임별 밝기 데이터까지 나머지 세 가지 신호를 추출합니다.

② 배치 검증 — 후보를 만들기 전에 컷 전체를 미리 확인합니다

첫 번째 단계에서 찾은 "장면 전환 후보" 전체를 마커를 만들기도 전에 한 번에 몰아서 AI가 컷 전후 화면 두 장을 비교해 실제로 다른 장면인지 확인합니다. 단순히 카메라 앵글이 바뀌거나 자막·그래픽이 들어온 것은 걸러냅니다. 동시에 컷 전후의 발화 내용을 읽어서 이야기 주제가 바뀌었는지도 파악합니다. 이 결과는 다음 ③ 후보 생성 단계에서 그대로 재사용됩니다 — 여기서 이미 "진짜"로 확인된 컷은 나중에 다시 확인하지 않습니다.

자세한 기술 정보 보기

장면 비교 AI (CLIP)

OpenAI의 이미지 이해 AI입니다. 컷 전후 프레임 두 장을 비교해 같은 장면인지 다른 장면인지를 0~1 유사도로 수치화합니다. 0.80 미만이면 진짜 장면 전환으로 확정합니다.

한국어 이해 AI (ko-sroberta)

한국어 특화 언어 모델입니다. 두 문장을 읽고 내용이 얼마나 비슷한지 수치화합니다. 0.75 미만이면 주제가 바뀐 것으로 판단해 점수에 반영합니다.

픽셀 비교 vs CLIP — 왜 두 가지를 같이 쓰나요?

PySceneDetect(픽셀 비교)는 인접한 두 프레임의 RGB 픽셀 값 차이를 수치로 계산합니다. 전체적으로 픽셀이 많이 바뀌면 장면이 전환됐다고 판단합니다. 계산이 단순해서 빠르지만, 조명이 갑자기 바뀌거나 카메라가 빠르게 흔들려도 장면 전환으로 잘못 잡히는 경우가 있습니다.

CLIP은 이미지와 텍스트를 수억 장 함께 학습해서 "이 화면이 어떤 장면인지"를 이해하는 딥러닝 모델입니다. 픽셀이 많이 달라져도 같은 장소·인물이면 전환이 아닌 것으로 걸러냅니다.

이 프로젝트에서 두 가지를 함께 쓰는 이유가 바로 이겁니다. PySceneDetect로 빠르게 후보를 좁히고, CLIP으로 그 후보가 진짜 장면 전환인지 정밀하게 검증합니다. 속도와 정확도를 동시에 잡는 구조입니다.

③ 후보 생성 — 세 가지 상황에서 마커를 찍습니다

아래 세 가지 상황 중 하나라도 해당되면 마커 후보가 생성됩니다. 중복되는 지점은 자동으로 하나로 합칩니다.

Path 1
말이 끊기는 순간
문장이 마무리되고 잠깐 조용해지는 구간입니다. 강의, 토크쇼, 예능처럼 말이 중심인 콘텐츠에서 가장 많이 잡힙니다.
  • ✓문장이 자연스럽게 끝나고
  • ✓0.5초 이상 음성이 없고
  • ✓그 침묵 안에 허용 프레임이 있을 때
Path 2
화면이 확 바뀌는 순간
AI가 확인한 진짜 장면 전환 지점입니다. 빠르게 편집된 영상에서 침묵이 없어도 잡아냅니다.
  • ✓AI가 진짜 장면 전환으로 확인하고
  • ✓전환 지점 근처에 문장 경계가 있고
  • ✓해당 프레임이 허용 프레임일 때
Path 3
화면이 어두워지는 순간
페이드 아웃으로 암전되는 구간입니다. 영화·음악 콘텐츠처럼 장면 사이에 페이드가 자주 쓰이는 영상에서 효과적입니다.
  • ✓화면이 서서히 어두워지는 구간이고
  • ✓가장 어두운 프레임에 허용 프레임이 있을 때
  • ✓음성 침묵 동반 여부는 장르별 설정 (영화·드라마·케이팝은 배경음이 이어져도 통과)
  • ✓같은 지점에 컷(Path 2)과 페이드가 동시에 걸리면 페이드가 우선 — 기존 컷 마커를 페이드 마커로 승격시키고, 컷에서 확인된 근거(CLIP 유사도 등)는 그대로 유지

④ 개별 재검증 — 애매하게 남은 소수만 한 번 더 확인합니다

② 배치 검증은 본편 구간(시작·끝 데드존 제외) 안의 컷만 미리 확인합니다. 그 범위 밖에 걸쳐 있거나 화면 추출에 실패해 값이 안 나온 극소수 컷은 ③에서 일단 "전환 후보"로 마커가 만들어진 뒤, 이 단계에서 개별로 한 번 더 CLIP 확인을 거칩니다.

여기서 "장면 전환이 아니다"로 뒤집히면 마커 자체를 지우지는 않습니다 — 문장이 끝나고 침묵이 있었다는 근거는 여전히 유효하기 때문입니다. 대신 ③에서 미리 더했던 장면 전환 점수를 그대로 빼서 "검증전환"에서 "참고" 마커로 낮춰 표시합니다.

대부분의 마커는 ②에서 이미 결론이 나 있어서, 이 단계를 거치는 건 소수입니다.

두 종류의 XML이 만들어집니다

1차 XML _adbreaks.xml

간격 규칙이 적용된 실사용 파일입니다. 첫 광고는 3~10분 사이, 이후 광고는 10~15분 간격으로 배치됩니다. 같은 구간 안에서 점수 순으로 최대 5개 후보가 표시됩니다.

2차 XML _adbreaks_all.xml

장면 전환·페이드가 확인된 모든 지점을 간격 제한 없이 담은 파일입니다. 1차 파일을 검토할 때 보조 참고용으로 사용합니다.

공유 서버 연결하기

먼저 와이파이 NEWID_Traffic_5G 에 연결되어 있어야 합니다. 다른 네트워크에 있으면 서버가 보이지 않습니다.

아래 순서대로 진행하세요.

  1. Finder를 엽니다
  2. ⌘ + K (이동 → 서버에 연결)
  3. 주소 입력 후 연결:
    smb://192.168.17.128
  4. 로그인 창이 뜨면 아래 계정으로 접속:
    이름(계정): guest_co
    비밀번호: 1111

연결되면 Public → Drop Box 경로로 들어가면 장르별 폴더가 보입니다.

2

장르 폴더에 영상 넣기

분석할 영상의 장르에 맞는 폴더에 파일을 복사하세요. 폴더마다 해당 장르에 최적화된 분석 기준이 자동으로 적용됩니다.

Drop Box/
   영화/  ← 장면 전환·페이드 중심
   드라마/  ← 장면 전환·대화 균형
   푸드/  ← 요리 시연·토크 균형
   케이팝/  ← 무대 전환 중심
   토크예능/  ← 발화·장면 전환 균형
   강의정보/  ← 주제 전환·챕터 구분
   여행/  ← 컷·침묵이 겹치는 지점
   자취남/  ← 일상 브이로그·주제 전환 중심
영상이 완전히 복사된 뒤 분석이 시작됩니다.
3

자동 분석 대기

파일이 감지되면 서버 컴퓨터가 자동으로 분석을 시작합니다. 영상 길이에 따라 수 분~수십 분이 소요됩니다.

4

결과 파일 확인

분석이 완료되면 같은 장르 폴더 안에 XML 파일 두 개가 생성됩니다.

드라마/
   YBJ_S25_EP01.mp4  ← 원본 영상
   YBJ_S25_EP01_adbreaks.xml  ← 1차 (실사용)
   YBJ_S25_EP01_adbreaks_all.xml  ← 2차 (전체 후보·참고용)

1차 _adbreaks.xml

간격 규칙(첫 광고 3~10분, 이후 10~15분)이 적용된 실사용 파일. 보통 이 파일을 사용합니다.

2차 _adbreaks_all.xml

장면 전환·페이드가 확인된 모든 지점을 간격 제한 없이 담은 파일. 1차를 검토할 때 보조 참고용입니다.

Premiere Pro → 파일 → 가져오기 → XML 선택 → 타임라인에 마커 자동 표시

각 장르 폴더 안의 settings.json 파일에 분석 기준이 저장되어 있습니다. 기준은 genres.json을 수정하면 모든 폴더에 일괄 반영됩니다.
장르별 기준 — 담당자가 알려주실 내용

이 도구는 장르마다 "광고를 어디에 넣는지" 규칙이 다릅니다. 각 장르 담당 편집자가 아래 항목을 알려주시면, 그 답을 분석 가중치로 변환해 장르에 맞게 조정합니다.

담당자가 알려주실 내용 (질문)
영상 시작 후 몇 분 안에는 광고를 넣지 않나요?
영상 끝 몇 분 안에는 광고를 넣지 않나요?
첫 광고는 보통 몇 분 ~ 몇 분 사이에 들어가나요?
광고와 광고 사이 간격은 몇 분 정도인가요?
이 장르에서 광고를 넣는 1순위 신호는? (화면 전환 / 이야기 주제 변화 / 화면 암전 / 말 끊김)
화면이 서서히 어두워지는 페이드(암전) 장면이 자주 나오나요? 그때 소리도 같이 사라지나요, 배경음이 이어지나요?
주로 마커를 찍는 지점의 특징에 대해 알려주세요.
