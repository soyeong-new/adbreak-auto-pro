# 마커 생성 · 가중치 설계 문서

> 작성 중. 결정된 사항을 단계별로 기록. 끼워맞추기 방지를 위해
> "측정 기반"과 "임시값"을 구분해서 표기한다.

## 설계 순서

1. 패스 구조 확정 ← **진행 중**
2. 점수 항목 목록 확정 (고정 vs 장르 가변)
3. 장르 파라미터 목록 + 단위 통일
4. 장르 프리셋 수치 + 검증상태 메타

## 1단계 — 패스 구조

### 유지
- Path 1(침묵 앵커), Path 2(컷 앵커), Path 3(페이드 앵커) **3개 모두 유지**

### 결정 사항

| 항목 | 결정 | 근거 |
|---|---|---|
| Path 3 침묵 관문 | **장르 파라미터화** (`fade_require_silence`) | KTA(영화) 13개 페이드 중 12개가 음악 지속 → 침묵 요구 시 영화 신호 죽음 |
| └ 영화·케이팝 | 침묵 **불필요** (페이드만으로 통과) | 페이드 위 배경 스코어 지속 |
| └ 나머지 장르 | 침묵 **필수** (현행 유지) | 검증된 현행 동작 보존 |
| 케이팝 침묵 보너스 | **케이팝만** "침묵 있으면 가산" | 다른 가수 등장 시 페이드인 → 침묵 동반이 강한 신호 |
| 컷 vs 페이드 중복(0.3s 이내) | **페이드 우선 + 증거 병합** | 영화에선 페이드가 챕터 경계로 더 중요 |
| └ 병합 방식 | 마커 1개로 합치되 페이드 플래그 + 컷의 CLIP 검증 + 텍스트 유사도 모두 보존 | 정보 손실 없음 |

### 미해결 / 확인 필요
- 케이팝 침묵 보너스 **계수 크기** → 데이터 부족, 임시값 후 측정.

## 3단계 — 장르 파라미터 + 단위

### genres.json 최종 파라미터 목록

| 키 | 단위/형 | 범위 | 설명 |
|---|---|---|---|
| `first_min` / `first_max` | 분 | — | 첫 광고 삽입 구간 |
| `gap_min` / `gap_max` | 분 | — | 광고 간 간격 |
| `w_scene` | 점수 | 0~10 | 장면 전환 가중치 (Path 1,2) |
| `w_topic_change` | 점수 | 0~10 | 주제 전환 가중치 (Path 1,2) |
| `w_fade` | 점수 | 0~10 | 페이드 전환 가중치 (Path 3) |
| `silence_min` | 초 | 0.1~1.0 | Path 1 침묵 최소 길이 |
| `fade_require_silence` | bool | — | Path 3 침묵 필수 여부 (영화·케이팝=false) |
| `fade_silence_bonus` | 점수 | 0~ | Path 3 침묵 동반 시 가산 (케이팝만 >0, 나머지 0) |
| ~~`p_cta`~~ | — | — | **삭제됨** |

### 단위 정책 — "사람 단위로 통일"

- genres.json은 **사람이 읽는 단위**(분·초·점수)로만 저장.
- 변환은 **코드 한 곳에서만**:
  - 분→초 (`first_min` 등): app.py에서 ×60 (현행 유지)
  - `silence_min`: **초 그대로** — UI의 ×10 변환 **제거**
- UI 침묵 슬라이더: min=0.1 max=1.0 step=0.1, 값에 "초" 붙여 표시. 내부 ×10 없음.

### 영향
- index.html `selectGenre`의 `sm: Math.round(silence_min*10)` 매핑 제거 → 직접 사용.
- 침묵 슬라이더 HTML(min/max/step/표시) 수정 필요.

## 4단계 — 프리셋 수치 + 검증상태 메타

### 신규 파라미터 장르별 값

| 장르 | fade_require_silence | fade_silence_bonus | 검증상태 |
|---|:---:|:---:|---|
| 영화 | `false` | 0 | 임시 — KTA(영화) 1편 관찰 |
| 드라마 | `false` | 0 | 임시 — 추정(데이터 없음) |
| 케이팝 | `false` | 2.0 | 임시 — 미측정 |
| 푸드 | `true` | 0 | 현행 검증됨 |
| 토크예능 | `true` | 0 | 현행 검증됨 |
| 강의정보 | `true` | 0 | 현행 검증됨 |
| 여행 | `true` | 0 | 현행 검증됨 |
| 자취남 | `true` | 0 | 현행 검증됨 |

> 기존 값(`w_scene`/`w_topic_change`/`w_fade`/`silence_min`)은 이번 설계에서
> 변경하지 않음. 새 파라미터 2개만 추가.

### 검증상태 메타 — genres.json 안에

각 장르에 `_검증상태` 객체를 둔다. 값 옆에서 바로 "임시/측정됨"을 알 수 있게.

```json
"_검증상태": {
  "fade_require_silence": "임시 — KTA(영화) 1편 관찰, 추가 데이터 필요",
  "fade_silence_bonus": "임시 — 미측정"
}
```

현행 검증된 장르는 `_검증상태` 생략 또는 "현행 유지"로 표기.

## 새 장르 데이터가 들어오면 — 값 수정 절차

이 도구로 끼워맞추기 없이 값을 갱신하는 방법:

1. **정답 입력**: 새 채널의 광고 시각을 `ground_truth.txt`에 적고
   `eval/load_ground_truth.py`로 `eval/ground_truth.json` 갱신.
2. **캐시 확보**: 해당 영상을 한 번 분석해 `.cache/`에 5종(transcript/scenes/
   voice/clip_cuts/text_sim) + fades 생성.
3. **측정**: `eval/measure_regression.py`를 새 채널에 맞게 돌려
   값 변경 전후 정답 재현을 비교. **재현이 오르는 방향으로만** 값 조정.
4. **반영**: `genres.json`의 해당 장르 값만 수정 + `_검증상태`를 "측정됨"으로 갱신.
   UI·공유폴더는 자동 반영 (setup_dropbox.sh 재실행).

> 핵심: 값은 **항상 정답 측정으로** 정한다. 직관/임시값은 `_검증상태`에 "임시"로
> 남겨, 나중에 측정값으로 교체할 대상임을 표시한다.

## Path 1 주제전환 + 자취남(JCN) 조사 결과 — 2026-06

### 결정: Path 1 주제전환 가점 **추가하지 않음 (보류)**

근거 (정답 측정):
- YBJ(토크) 21편: 적용 시 1차 재현 2→3 (+1), 전체 34→34. 19/21편 추천 재정렬.
- JCN(자취남) 14편: 적용 시 1차 1→2 (+1), 전체 7→7.
- 변동 대비 이득이 미미. 끼워맞추기 위험 > 이득.

### 자취남의 진짜 문제 — 후보 생성 한계 (조사 결과)

`eval/diagnose_jcn_misses.py`, `eval/explore_jcn_signal.py`로 측정:

- 자취남 정답의 **63%(12/19)가 후보로 안 잡힘**. 전체 후보 재현 천장 = **37%**.
- 놓친 지점 특징: 문장경계 있음(11/12) · 침묵 없음(11/12) · 컷 없음 · 페이드 없음.
  즉 "쉬지 않고 말이 이어지는 문장경계"라 Path 1(침묵 요구)이 못 잡음.
- 주제전환(<0.75)은 **변별력 없음**: 전체 문장경계의 94%가 해당.
  정답 vs 일반 경계 유사도 분포 거의 동일(중앙 0.59 vs 0.62).
  침묵·문장길이·간격도 분리 안 됨. 국소최저점만 4배(7.9% vs 1.9%)이나 절대값 낮음.

**결론**: 현재 피처(임베딩 유사도·침묵·문장구조)로는 자취남 광고점을 일반 경계와
구분 불가. 튜닝 문제 아님 — 근본적 신호 부재. 자취남은 **기존 기준(침묵·컷·페이드)
그대로 사용, 37% 천장 수용**. 더 올리려면 로컬 LLM 챕터 분할 등 별도 아키텍처 필요(미정).

> 옵션2 변경은 자취남 후보 생성에 영향 없음 — 침묵 경로 생성 조건 불변,
> fade_require_silence=true 유지. 37%는 29.97 시절과 동일 검출력.
